# US-001-建立待辦事項

**所屬 Epic：** [[EPIC-001-待辦事項管理]]

## 故事

身為 TaskFlow 使用者，
我想要輸入標題後建立一筆待辦事項，
以便把要做的事情記錄下來、不再依賴聊天訊息。

## 驗收條件 (Acceptance Criteria — Given/When/Then)

- **Given** 使用者在待辦事項清單頁面
  **When** 使用者輸入標題並送出
  **Then** 清單中立即出現這筆新的待辦事項，狀態為「未完成」

- **Given** 使用者沒有輸入任何標題
  **When** 使用者按下送出
  **Then** 系統不建立事項，並提示標題為必填

## 邊界情況 (Edge Cases)

- 標題只有空白字元 → 視為未填寫，拒絕建立
- 標題超長（> 200 字）→ 由前端與後端各自截斷/拒絕，見 [[API Spec]] 的錯誤情況
