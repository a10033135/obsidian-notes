# 00-Meta

跨角色共通規範的存放處。這裡不放任何單一功能的規格內容，只放「所有人都要遵守的規則」與「產出新文件時要用的樣板」。

## 內容

| 檔案/資料夾 | 用途 |
|---|---|
| [[共通規範]] | 文件命名規則、狀態標籤定義、SDD 工作流程 |
| [[Tags 說明]] | Obsidian tag 分類系統，用於跨文件檢索 |
| `Templates/` | 每一種文件類型的樣板，搭配 Obsidian 的「範本」核心外掛或 Templater 外掛使用 |

## 何時更新這個資料夾

- 新增一種新的文件類型（例如新增「Postmortem」文件）時，先在這裡新增樣板與規範，再去對應角色資料夾建立實例。
- 修改任何樣板前，請先確認不會破壞既有文件的一致性（例如欄位改名）。
